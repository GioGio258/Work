# 实验三：贝塞尔曲线光栅化

## 实验目标

1. 理解光栅化的基本概念，手动"点亮"屏幕上的像素
2. 掌握 De Casteljau 算法，用于计算贝塞尔曲线上的点
3. 学习使用 Taichi 的 GPU 并行计算能力，实现高效的曲线绘制
4. 了解 CPU 与 GPU 之间的数据传输优化（Batching 技术）

## 代码逻辑

### 1. 初始化与显存预分配

- **常量定义**：
  - `WIDTH = 800`, `HEIGHT = 800`：屏幕尺寸
  - `MAX_CONTROL_POINTS = 100`：最大控制点数
  - `NUM_SEGMENTS = 1000`：曲线采样数
- **GPU 缓冲区**：
  - `pixels`：800x800 大小的像素缓冲区，用于存储最终画面
  - `gui_points`：控制点数据缓冲池
  - `gui_indices`：控制多边形索引
  - `curve_points_field`：曲线坐标缓冲区，用于一次性接收 CPU 算好的曲线坐标

### 2. De Casteljau 算法

- **de\_casteljau(points, t)**：
  - 功能：使用 De Casteljau 算法计算参数 t 对应的曲线上的点
  - 实现：递归地在每两个相邻控制点之间插值，直到得到最终的曲线上的点
  - 参数：
    - `points`：控制点列表
    - `t`：参数，范围 \[0, 1]

### 3. GPU 绘制内核

- **clear\_pixels()**：
  - 功能：并行清空像素缓冲区
  - 实现：使用 `@ti.kernel` 装饰器，在 GPU 上并行执行
- **draw\_curve\_kernel(n)**：
  - 功能：在 GPU 上并行绘制曲线
  - 实现：
    1. 从 `curve_points_field` 中读取浮点坐标
    2. 将坐标映射为整数像素索引（乘以宽高）
    3. 给像素缓冲区对应的位置赋绿色
    4. 进行越界检查（0 <= x < 800, 0 <= y < 800）

### 4. 主循环与交互响应

- **main()**：
  - 创建 GUI 窗口（800x800）
  - 处理鼠标和键盘事件：
    - **鼠标左键点击**：添加控制点
    - **按 'c' 键**：清空控制点
  - 计算与绘制逻辑：
    1. 在 CPU 端循环 1000 次，算出所有曲线上的点
    2. 使用 `curve_points_field.from_numpy()` 批量拷贝到 GPU
    3. 调用 `draw_curve_kernel()` 让 GPU 画图
    4. 将 `pixels` 传给 `canvas.set_image()` 显示
  - 绘制交互控制点（对象池技巧）：
    1. 创建长度为 100 的 NumPy 数组，全部填充为 -10.0（藏在屏幕外面）
    2. 把真实的控制点覆盖到数组的前几个位置
    3. 将其 `from_numpy` 给 `gui_points` 并进行绘制

## 注意事项

- **后端选择**：如果 GPU 初始化失败（例如找不到 Vulkan 着色器文件），可以将 `ti.init(arch=ti.gpu)` 改为 `ti.init(arch=ti.cpu)`
- **性能优化**：使用 Batching 技术，在 CPU 里把 1000 个点的坐标全算好，装进一个数组，一次性发给 GPU，然后让 GPU 并行地把这 1000 个像素点亮
- **对象池技巧**：绘制辅助线时，预先分配一个固定大小的显存池，不要传给它长度忽大忽小的数组

## 实验原理

### 1. 光栅化基础

屏幕本质上是一个巨大的二维像素网格。在本实验中，需要手动"点亮"对应的像素：

- 创建一个尺寸为 800x800 的二维数组（在 Taichi 中称为 Field），用来模拟屏幕显存
- 这个数组存储了每个像素的 RGB 颜色值
- 将浮点坐标乘以屏幕的长宽，并强制转换为整数（int），从而找到它在二维数组中对应的物理像素索引

### 2. De Casteljau 算法

De Casteljau 算法是一种计算贝塞尔曲线上点的递归方法：

- 对于给定的参数 t（范围 \[0, 1]），在每两个相邻控制点之间插值
- 得到新的控制点列表，重复这个过程
- 最后得到的点就是曲线上对应 t 值的点

### 3. 性能优化（Batching 技术）

在现代图形学编程中，CPU 和 GPU 是物理分离的，它们之间通过 PCIe 总线通信：

- 如果在 Python（CPU）的 for 循环里，每次算出一点就去修改 Taichi 的 GPU Field，海量的跨界通信将使帧率大幅降低，造成卡顿
- 正确的做法：在 CPU 里把 1000 个点的坐标全算好，装进一个数组，一次性发给 GPU，然后让 GPU 并行地把这 1000 个像素点亮

### 4. 对象池技巧

由于 `canvas.circles()` 只能接收定长的 Taichi Field：

- 在 Python 中创建一个长度为 100 的 NumPy 数组，全部填充为 -10.0（藏在屏幕外面，GPU 画了也看不见）
- 然后把真实的控制点覆盖到数组的前几个位置
- 最后将其 `from_numpy` 给 `gui_points` 并进行绘制
